# Обмен данными `images` ↔ `preview` у Product (продакшн)

## Зачем

На **деве** база может быть пустой, а реальные картинки — на **проде**. Миграция `database/migrations/2026.02.10T00.00.00.swap-product-images-preview-fields.js` выполняется **автоматически один раз** при первом запуске Strapi после деплоя (до синхронизации схемы с content-types).

Она **только переставляет** значения колонки `field` в morph-таблице связей файлов с продуктами (`images` ⇄ `preview`), не трогая сами файлы в Media Library.

Имя таблицы у Strapi 5 может быть **`files_related_morphs`** или сокращённое **`files_related_mph`** — миграция учитывает оба варианта.

Схема `product` уже корректна: `images` — несколько файлов, `preview` — один. Меняются **привязки**, чтобы «главная» картинка оказалась в `preview`, а галерея — в `images`.

## Перед деплоем на прод

1. **Полный бэкап БД** (дамп PostgreSQL / MySQL или копия SQLite).
2. Задеплойте версию с этой миграцией и обновлённым кодом API/фронта (если вы уже опираетесь на `preview` + `images` в ответах).
3. Запустите Strapi — в логах не должно быть ошибки миграции.
4. Проверьте в админке несколько продуктов: в **Preview** должна быть бывшая одиночная картинка из старого `images`, в **Images** — бывший `preview` (галерея).

## Если миграция не нашла таблицу

Редко имя таблицы может отличаться (очень длинные имена, кастомный префикс). Тогда:

- Посмотрите в БД таблицу вида `*related*morph*` с колонками `field`, `related_type`.
- При необходимости поправьте константу `DEFAULT_TABLE` в файле миграции под ваш прод.

## Откат

Штатного `down` нет. Откат = **восстановление бэкапа БД** или повторный обмен `images`/`preview` тем же SQL/Knex-логикой (второй раз миграция не выполнится — запись уже в `strapi_migrations`).

## Дев без данных

На пустой базе morph-таблицы может ещё не быть: миграция **всё равно помечается выполненной** в `strapi_migrations`. Если потом вы **зальёте дамп с прода** в эту же дев-базу, swap уже не произойдёт автоматически.

**Что делать:** либо импортируйте прод-дамп **до** первого запуска этой версии Strapi, либо после импорта удалите запись о миграции `2026.02.10T00.00.00.swap-product-images-preview-fields` из `strapi_migrations` и перезапустите Strapi один раз, либо выполните тот же swap вручную SQL/Knex по таблице `files_related_morphs`.

## Прод (основной сценарий)

На проде база уже существует, таблица связей есть — при деплое миграция отработает **один раз** и переставит `field` для всех продуктов. Делайте **бэкап перед деплоем**.
